# Глава 4: На стыке миров

## Ключевая идея
Проблемное и проектное обучение — подходы, соединяющие технологичность результато-ориентированного проектирования с ориентацией на человека. Их основа — «обучение целому больше, чем сумма частей» и аутентичные (реальные) задачи.

## Введённые фреймворки

- **Проблемное обучение (problem-based learning, PBL)** — самостоятельное познание через столкновение с аутентичными проблемами (корни: Дьюи; медшкола McMaster). Целостное обучение комплексному навыку. Требует редизайна не только «как», но и «чему» учим (кросс-предметность). Сильная **доказательная база** (отличие от классических теорий-наблюдений).

- **Четырёхкомпонентная модель 4C/ID (Й. ван Мерриенбур)** — самая техничная/системная, для комплексного обучения. 4 компонента + 10 шагов:
  - **Учебные задачи** (скелет): проектирование задач → методов оценки деятельности → последовательности.
  - **Поддерживающая информация**: проектирование → анализ когнитивных стратегий → анализ ментальных моделей.
  - **Процедурная информация**: проектирование → анализ когнитивных правил → анализ пререквизитов.
  - **Частичная практика**: тренировка группы умений до автоматизма.
  - Принципы учебной задачи: целостная, аутентичная, комплексная, модифицируемая.
  - Структура задачи: вводные + цель + подход к решению + процесс решения. Варьируя компоненты → пример, имитация, обратный кейс, задача без цели и т.д. (Приложение 6).
  - **TOSC** (аналог ABCD для 4C/ID): tools & objects + standards + condition.
  - Поддерживающая информация = ментальная модель предметной области; процедурная = пошаговая инструкция «если — то».
  - Динамика: больше поддержки → меньше поддержки; от простого к сложному. Минус: высокая трудозатратность.

- **Проектное обучение (project-based learning, PjBL)** — «learning by doing» (Дьюи, Килпатрик «Проектный метод» 1918). Универсальные ингредиенты: цели на глубокое понимание + применение; сложная вовлекающая задача; аутентичность; непрерывное исследование; право голоса/выбора студента; рефлексия; обратная связь и модернизация; публично представленный продукт.
  - **7 этапов**: 1) знакомство с целями (для старшей школы — сразу критерии); 2) предварительное погружение; 3) формулирование задачи через исследование; 4) генерация решения (брейншторм, ТРИЗ); 5) реализация (итерационно); 6) презентация + неструктурированная ОС; 7) рефлексия + критериальная ОС.
  - Проблемное обучение ДО проектного → лучший результат (Стэнфорд + Вандербильт).

- **Модели обучения (по М. Кларину)** — целостный блок процесса с целью + схемой организации деятельности. Три группы:
  - **Исследовательские (discovery learning)** → на практике **направляемое (guided discovery)**: 1) создание исследовательской проблемы; 2) проектирование поддержки (scaffolding). Вариации: **обучение через вызов** (challenge-based: большая идея → вопросы → вызов → решение → оценивание → публикация), **обучение через интерес** (inquiry-based, фокус на постановке студентом своего вопроса, Приложение 7).
  - **Обучение через пространство (place-based)** — новое пространство ломает паттерны мышления; изучение локальных сообществ (THINK Global School, Minerva).
  - **Игровое обучение.**

## Ключевые понятия
- **Skills / skillset** — выполняемые действия / иерархия выполняемых действий.
- **Когнитивные правила** — цепочка «если — то», запускается автоматически.
- **Пререквизиты** — знания для корректного выполнения «если — то».
- **Частичная практика** — отработка декомпозированных умений до автоматизма.
- **Scaffolding** — поэтапная поддержка, убираемая по мере роста студента.

## Различие целеполагания
- Результато-ориентированный: что человек **научится** (узнает, поймёт, проанализирует).
- Проблемно-ориентированный: что человек **сделает** (отправная точка — учебная задача/проблема, а не результат).

## Анти-паттерны
- **Атомарное проектирование через ОР** — теряется взаимосвязь элементов комплексного навыка.
- **Бросать студента сразу на большой комплексный вызов** без серии тренировочных задач.
- **Чистое (ненаправляемое) discovery learning** — студенты не приходят к нужным выводам; нужно guided.

## Ключевые выводы
1. Для комплексного навыка — 4C/ID: сначала скелет учебных задач, потом обрастание информацией.
2. Модифицируй одну задачу по параметрам (вводные/цель/решение), уменьшая поддержку.
3. Проектное обучение = конструктивная согласованность между целями, проектной деятельностью и критериальным оцениванием.
4. Исследовательские модели работают только как направляемые (guided + scaffolding).

## Связи
