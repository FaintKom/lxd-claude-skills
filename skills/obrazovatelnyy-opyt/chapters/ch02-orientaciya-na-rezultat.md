# Глава 2: Ориентация на результат

## Ключевая идея
Результато-ориентированное обучение строит триаду **образовательный результат → учебная стратегия → метод оценки** и требует их **конструктивной согласованности**. Сдвиг фокуса: не «что студент изучит», а «что он сможет делать».

## Введённые фреймворки

- **Образовательный результат (learning outcome)** — описание наблюдаемого/измеримого действия после обучения. 6 параметров (Дж. Гилберт, ~SMART): релевантный, недвусмысленный, выполнимый (в зоне ближайшего развития), непротиворечивый, наблюдаемый, измеримый.

- **Формула ABCD (Р. Магер)** — конструктор образовательного результата:
  - **A (audience)** — аудитория (кто учащийся).
  - **B (behavior)** — поведение (наблюдаемое действие; в одном из доменов Блума).
  - **C (condition)** — условие (при каких обстоятельствах/инструментах).
  - **D (degree)** — степень (золотой стандарт: время, точность, качество).
  - Когда использовать: для точного описания ОР. Риск: грань между ОР и методом оценки стирается.

- **Таксономия Блума (ред. Кратволь–Андерсон, 2001)** — двухмерная. Уровни освоения (снизу вверх): **Знает → Понимает → Применяет → Анализирует → Оценивает → Создаёт (синтез)**. Второе измерение — типы знания: фактические, концептуальные, процедурные, метакогнитивные. Один ОР = пересечение «уровень × тип знания». Каждый уровень — свой набор глаголов (см. Приложение 1).

- **Таксономия Марцано (1998, evidence-based)** — отказ от иерархии когнитивных операций; иерархия между тремя доменами: **когнитивная система → метакогнитивная система → я-система**. Уровни 1–6: воспроизведение → понимание → анализ → использование знания → метакогнитивный → я-система. Смещает фокус от активностей к человеку и его восприятию (важность, эффективность, эмоции, мотивация). Когда использовать: когда нужна субъектность и метапознание.

- **Методы оценивания** — три цели:
  - **Оценивание ДЛЯ обучения** (for learning) — диагностика (входное тестирование, наблюдение, портфолио-ревью), корректирует программу.
  - **Оценивание КАК обучение** (as learning) — формирует знание о себе или содержании (игровой тест с подсказками).
  - **Оценивание обучения** (of learning) — делится на **формирующее** (formative, «в моменте», гибкость) и **суммирующее/итоговое** (summative, финальный результат: экзамен, performance assessment, proficiency).

- **Конструктивная согласованность (Дж. Биггс)** — треугольник: образовательные результаты ↔ учебные стратегии ↔ методы оценки логически вытекают друг из друга. + Заложить пространство для самостоятельного конструирования смысла студентом. Реакция на **атомарность** проектирования (невозможно удержать сотни ОР в голове).

- **Учебные стратегии (Conversational Framework, Д. Лорильяр)** — 6 категорий методик: **приобретение** (лекция, чтение), **сотрудничество** (групповая работа, peer-to-peer), **дискуссия** (дебаты, кейс), **исследование** (inquiry, доклад), **практика** (упражнения), **создание** (проект для реального заказчика). Сила фреймворка — сбалансированность: вариативность методик из разных категорий.

- **Обратный дизайн / Understanding by Design (UbD, Виггинс–Мактай)** — 3 этапа: 1) определение желаемых результатов (цели на перенос, большие смыслы, основополагающие вопросы); 2) определение необходимых свидетельств (как поймём, что случилось); 3) планирование учебного опыта. UbD добавляет к backward design более абстрактные смыслы и перенос в жизнь.

## Ключевые понятия
- **SME (subject matter expert)** — предметный эксперт, наполняет ОР содержанием.
- **Критериальное оценивание** — суждение по заранее заданным критериям (рубрикатор); даёт траекторию уровней. Контраст — **нормативное** (сравнение со среднестатистической нормой; ЕГЭ, SAT).
- **Рубрикатор** — документ с уровнями освоения ОР; без абстрактных прилагательных (хороший/плохой).
- **Обратная связь (feedback)** — указать проблемную зону + подсказать стратегию исправления.

## Анти-паттерны
- **Расплывчатые формулировки ОР** («будет знать») — без свидетельства непроверяемо (Дж. Мун).
- **Описывать процесс обучения вместо результата.**
- **Атомарное проектирование** — ОР, методики и оценка не согласованы между собой.
- **Сразу строить гипотезы об оценке** при формулировке ОР → фокус смещается на аттестацию, а не на опыт.

## Ключевые выводы
1. Описывай ОР как наблюдаемое действие; ABCD — рабочий конструктор.
2. Выбирай таксономию: Блум — для уровней освоения когнитивного; Марцано — когда нужны метапознание и субъектность.
3. Проверяй конструктивную согласованность как чек-лист: ОР ↔ методика ↔ метод оценки.
4. Балансируй учебные стратегии из 6 категорий Лорильяр.
5. Обратный дизайн = «с конца»: результаты → свидетельства → опыт.
