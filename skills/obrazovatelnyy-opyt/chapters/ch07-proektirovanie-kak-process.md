# Глава 7: Проектирование как процесс

## Ключевая идея
**Алгоритм проектирования** (instructional design model) задаёт этапность проекта «создать опыт»; **подход к проектированию** (от результата/проблемы/опыта) указывает, как создать *определённый* опыт. Универсальный алгоритм School of Education соединяет оба домена.

## Введённые фреймворки

- **Организация содержания (последовательности, Дж. Познер)** — 5 кластеров:
  1. **Последовательность реального мира**: пространственная, временная/хронологическая, по физическим параметрам.
  2. **Концептуальная**: по классовой принадлежности, по связям между утверждениями (причина–следствие, теория–пример), по принципу усложнения (спираль), по логической предпосылке (A нужно для B — точные науки).
  3. **По тому, как содержание изучалось**: по логике исследования (форма рассуждения, индукция), по практике исследования (как у учёного).
  - Нужна, если: есть устоявшиеся взаимосвязи понятий И/ИЛИ освоение длительное.

- **ADDIE** (последовательный алгоритм, армия США, 1980-е): **A**nalyze → **D**esign → **D**evelop → **I**mplement → **E**valuate. Логика «закрытия разрыва» между текущим и желаемым → связан с проектированием от ОР.
  - На Design — конструктивная согласованность; в новых версиях — тестирование решения (прототип).
  - Develop часто = очная сессия по **9 событиям Ганье** (завладеть вниманием → цель → пререквизиты → содержание → поддерживаемая практика → независимая практика → обратная связь → оценить → завершить).
  - Evaluate часто по **Киркпатрику** (4 уровня: реакция → обучение → поведение → результаты) + ROI (Филипс).

- **Дизайн-мышление** (человекоцентричный, Стэнфорд/IDEO): исследование → интерпретация → генерация → эксперимент → развитие. Фокус не на программе, а на **закрытии потребности** аудитории. Инструменты эмпатии: интервью, наблюдение, шедоуинг. Вызов может касаться плана, пространства, процессов, системы.

- **Спиральная модель / 5D** (итеративный): **D**iscovery → **D**esign → **D**evelopment → **D**eliver → **D**ebrief. Старт сразу с гипотезы решения (из брифа), разворачивается по спирали, каждый виток уточняет 5 компонентов: учащийся, цели/результаты, стратегии, методы оценки, метрики. Аналог — SAM («прогрессивный джипег»).

- **Универсальный алгоритм проектирования (School of Education)** — 8 шагов:
  1. **Исследование и анализ** — 3 категории данных: учебные + сопутствующие характеристики; потребности/задачи учащихся и стейкхолдеров; релевантные примеры и контекст. → «рычаги влияния».
  2. **Организация содержания** (в паре с SME).
  3. **Определение образовательных целей** + выбор подхода к проектированию (взаимовлияют) → концепция программы.
  4. **Создание учебного плана** (конструктивная согласованность): соотнести содержание с уровнями освоения/типами задач, затем ОР ↔ стратегии ↔ методы оценки.
  5. **Проектирование опыта** — карта опыта; анализ когнитивной нагрузки, мотивации, динамики.
  6. **Проектирование среды** + **исследовательских интервенций** (критерии успеха: доходимость/completion rate, SSI) + создание методических/учебных материалов.
  7. **Анализ опыта** и сбор бэклога (анализ дизайна + анализ методматериалов).
  8. **Перепроектирование/корректировка** — дизайн воплощается на 3–4-м запуске; процесс цикличен (по спирали).

## Ключевые понятия
- **Алгоритм vs подход** — этапность проекта vs способ создания конкретного опыта.
- **Большие идеи** — организация содержания, сочетается с проблемным подходом.
- **Рычаги влияния** — выводы анализа для среды/методологии/содержания.
- **Доходимость (completion rate)**, **SSI (Student Satisfaction Index)**, **бэклог**.

## Выбор алгоритма
- Не ограничены, фокус на человеке → **дизайн-мышление**.
- Много идей, понятна аудитория, надо быстро → **итерационная (5D)**.
- Чёткий разрыв в знаниях/навыках, корпоративный контекст → **ADDIE**.

## Анти-паттерны
- **Размеренный переход шаг-за-шагом** — на практике шаги наслаиваются, результаты влияют на предыдущее.
- **Анализ только продуктовых параметров** (кто купит) вместо данных, влияющих на опыт.
- **Воспринимать учебный план как бюрократию** — это методическое ядро.

## Ключевые выводы
1. Различай алгоритм (как проектировать процесс) и подход (как создать конкретный опыт).
2. Организуй содержание осознанно (5 кластеров Познера) в паре с SME.
3. Универсальный алгоритм: 8 шагов, цикличность, воплощение на 3–4-м запуске.
4. Выбирай алгоритм под ситуацию (свобода/скорость/чёткость разрыва).
