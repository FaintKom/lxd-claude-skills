# Шпаргалка

## Какую таксономию выбрать
| Ситуация | Таксономия |
|---|---|
| Уровни освоения когнитивного, нужна простота | Блум (Знает→…→Создаёт) |
| Нужны метапознание, мотивация, субъектность | Марцано (когнитивный/метакогнитивный/я-система) |
| Цели на чувства, ценности, установки | Аффективный домен (Кратволь) |
| Один ОР через несколько уровней | Марцано (домены пересекаются) |

## Проблемное vs проектное обучение
| | Проблемное (PBL) | Проектное (PjBL) |
|---|---|---|
| Суть | последовательное решение серий задач | пошаговая разработка решения |
| Отправная точка | учебная задача/проблема | абстрактная проектная задача |
| Результат | решённые задачи | публичный продукт/артефакт |
| Связка | 4C/ID (задача+поддержка) | 7 этапов, критериальная ОС |
| Совет | хорошо ставить ДО проектного | давай серию тренировочных задач до большого вызова |

## Подход к проектированию
| Цель | Подход |
|---|---|
| Предсказуемый измеримый результат | от результата (ABCD, UbD, конструктивная согласованность) |
| Трансформация, личные смыслы | от опыта (цикл Колба) |
| Комплексный навык, реальные задачи | от проблемы (4C/ID) |
| Понимание + публичный продукт | проектное (PjBL) |
| Самостоятельное познание | исследовательское (guided discovery) |

## Выбор алгоритма проектирования
| Контекст | Алгоритм |
|---|---|
| Чёткий разрыв знаний, корпоратив | ADDIE |
| Свобода решений, фокус на человеке | дизайн-мышление |
| Много идей, понятна аудитория, скорость | итерационный (5D/SAM) |
| Соединить подход + этапность | универсальный алгоритм (8 шагов) |

## Виды оценивания
| Цель | Тип |
|---|---|
| Диагностика, корректировка программы | для обучения (for learning) |
| Знание о себе/содержании | как обучение (as learning) |
| Прогресс «в моменте», гибкость | формирующее (formative) |
| Финальный результат | суммирующее (summative): экзамен, performance assessment |
| По заданным критериям | критериальное (рубрикатор) |
| Сравнение с нормой | нормативное (ЕГЭ, SAT) |

## Конструктивная согласованность (чек-лист)
☐ ОР наблюдаем/измерим (ABCD)? ☐ Метод оценки соответствует ОР? ☐ Учебная стратегия ведёт к ОР? ☐ Стратегии из разных категорий (6 Лорильяр)? ☐ Есть пространство для своего смысла студента?

## 6 категорий методик (Лорильяр)
приобретение · сотрудничество · дискуссия · исследование · практика · создание → балансируй.

## Когнитивная нагрузка — баланс
Снижай **внутреннюю** (дели знание) + **внешнюю** (упрощай доставку): рабочий пример, убрать избыточное, всё вместе, модальность, визуализация. Эксперту → больше вызова. Замер: нагрузка = время (× пересмотр + конспект).

## Эмоциональная динамика
Активизирующее удовольствие (хочу скорее!) ✓ vs расслабляющее (слишком просто) ✗. Тренд должен колебаться (фрустрация → преодоление), но без пиков. Ровный позитив = поверхностный опыт.

## Цикл Колба (4 модуса + не по кругу)
CE (конкретный опыт) ↔ AC (абстрактная концептуализация) · RO (рефлексия) ↔ AE (эксперимент). НЕ начинай жёстко с CE. 9 стратегий = «как» действует студент.

## Цифровая среда — 3 ползунка
автономность ↑ → навигирующая/поддерживающая/прозрачная · размер ↑ → peer-to-peer · длительность ↑ → понижай автономность, усиль соцвзаимодействие.

## Здоровый климат (CSCI) — ключевое
**Безопасность** (#1) + поддерживающее обучение + уважение + самоидентификация + опрятность + инклюзивность + лидерство + отношения команды.

## Пять фаз / Восемь шагов
Фазы (Гл.1): анализ → концепция → опыт → производство → оценивание.
Универсальный алгоритм (Гл.7): исследование → содержание → цели+подход → учебный план → опыт → среда+интервенции+материалы → анализ → перепроектирование. Цикличен, воплощается на 3–4-м запуске.
