# Паттерны и техники проектирования

## Обратный дизайн (backward design / UbD)
**Когда**: проектирование от образовательных результатов.
**Как**: 1) определить желаемые результаты (цели на перенос, большие смыслы, основополагающие вопросы); 2) определить свидетельства достижения; 3) спланировать опыт под демонстрацию свидетельств.
**Trade-off**: предсказуемый результат, но слабая субъектность учащегося.

## Конструктивная согласованность как чек-лист
**Когда**: на этапе создания учебного плана.
**Как**: для каждого ОР проверь связку ОР ↔ учебная стратегия ↔ метод оценки; заложи пространство для самостоятельного конструирования смысла; сбалансируй методики из 6 категорий Лорильяр.
**Trade-off**: преодолевает атомарность, но требует системной работы и SME.

## Декомпозиция ОР по таксономии
**Когда**: описание уровней освоения.
**Как**: Блум — «уровень освоения × тип знания» (фактический/концептуальный/процедурный/метакогнитивный); Марцано — разложить по доменам (когнитивный → метакогнитивный → я-система), один ОР может охватывать несколько.

## Формула ABCD / TOSC
**Когда**: точное описание ОР (ABCD) или учебной задачи 4C/ID (TOSC).
**Как**: ABCD = аудитория + поведение + условие + степень. TOSC = tools & objects + standards + condition.
**Trade-off**: точность, но риск смешать ОР с методом оценки.

## Проектирование от опыта (цикл Колба)
**Когда**: опыто-ориентированное обучение.
**Как**: целеполагание (вектор) → ситуации опыта (симуляции/игры) → рефлексия (цикл Гиббса) → концептуализация (дискуссия + теория) → практика/закрепление. Не начинай жёстко с CE; давай входить через привычную стратегию.

## Конструктор симуляции (12 шагов)
**Как**: цель события → свидетельства изменений → способ сбора → мешающее поведение → целевое поведение → нужна ли игра и почему сейчас нет поведения → нужный опыт → сеттинг → роли → циклы «гипотеза–действие–фидбэк–коррекция» → рост сложности → правила принятия решений.

## 4C/ID: скелет учебных задач
**Когда**: комплексный навык, высокая трудозатратность оправдана.
**Как**: спроектировать пул учебных задач (целостных, аутентичных, модифицируемых) → методы оценки деятельности → последовательность (от простого к сложному, поддержка убывает) → поддерживающая информация (ментальные модели) → процедурная информация («если–то») → частичная практика.

## Управление когнитивной нагрузкой (7 стратегий)
**Как**: от существующих знаний; рабочий пример; рост независимости с экспертностью; убрать избыточное; всё важное вместе; модальность (устно+наглядно); побуждать визуализировать. Эффекты контекстуальны.
**Техника замера**: переводи нагрузку во время (видео × пересмотр + конспект).

## Карта опыта (Student Journey Map)
**Когда**: переход от учебного плана к опыту.
**Как**: слои — стадии (онбординг/основная/офбординг) → события глазами студента → когнитивная нагрузка (в минутах) → эмоциональная динамика (кривая по 5 параметрам) → мотивационная поддержка. Сначала гипотеза, потом инструмент анализа.

## Исследовательские интервенции
**Когда**: анализ опыта на запущенной программе.
**Как**: анкеты/поп-апы/интервью, проверяющие гипотезы карты; критерии успеха (доходимость, SSI); расставь «крючки» замера на карте → слои находки/боли/идеи → бэклог.

## Направляемое исследовательское обучение (guided discovery)
**Как**: 1) создать исследовательскую проблему (частично охватывает знания, мотивирует); 2) спроектировать scaffolding (доступ к знаниям, поддержка гипотез, планирование, пошаговое усложнение, связи через интеллект-карты).
**Trade-off**: лучше чистого discovery, но требует поддержки.

## Организация содержания (5 кластеров Познера)
**Как**: реальный мир (пространство/время/физпараметры); концептуальная (классы/связи утверждений/усложнение-спираль/логическая предпосылка); по тому, как изучалось (логика/практика исследования). Нужна при устоявшихся взаимосвязях И/ИЛИ долгом освоении.

## Универсальный алгоритм проектирования (8 шагов)
**Как**: 1) исследование/анализ (3 категории данных → рычаги влияния); 2) организация содержания (с SME); 3) цели + подход; 4) учебный план (согласованность); 5) проектирование опыта (карта, нагрузка/мотивация/динамика); 6) среда + интервенции + материалы; 7) анализ опыта/бэклог; 8) перепроектирование. Цикличен, воплощается на 3–4-м запуске.

## Установление раппорта
**Как**: называть по именам, хвалить, оперативно отвечать; юмор, неформальные диалоги, истории о себе; напоминания о дедлайнах, прозрачный план; уважение и эмпатия; указывать на сходство со студентами. Онлайн: вводный ролик, еженедельные публикации, персонализированные рассылки.
